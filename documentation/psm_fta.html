⚡ Critical Finding — Hardware FSM AND Gate Eliminates Independent Paths

The final stability decision in safety_isr.cpp is a logical AND: sw_ok (evaluate_stability, running on Teensy DTCM RAM) AND hw_gate (asm_fsm_read_stable on ASM330LHB embedded Cortex-M0+, independent silicon, independent power domain, independent RAM). SAFE_OUT_PIN goes HIGH only when both channels independently confirm stable. This eliminates all single-origin RAM corruption paths — a corrupted tilt buffer cannot produce FALSE HIGH because the hardware FSM provides an independent veto.

FSM_STABLE_NEED = 3: the hardware FSM counter must reach 3 consecutive stable evaluations (3 × 200 ms = 600 ms temporal hysteresis) before hw_gate asserts.

▲ DOMINANT RESIDUAL RISK — Correlated (Common-Mode) Sensor Failure

Both IMU SPI channels returning "stable" data simultaneously under common-mode EMI, supply voltage droop, or SPI bus corruption. This path fools both sw_ok (software Mahony AHRS) and hw_gate (on-chip FSM), because both consume the same physical sensor output. SPI config-register CRC-8 shadow implemented (poly=0x07, init=0xFF; covers CTRL1_XL / CTRL2_G / CTRL3_C / CTRL4_C; verified every 200 ms — 3 consecutive failures → FAULT_IMU_A/B_COMM). However, per-burst sensor data frame CRC (12-byte OUTX_L_G reads at 833 Hz) is not yet implemented — this remains a residual gap on the dominant path. Estimated PFH ≈ 10⁻⁵/hr → Serious (I-D).

4 Analysis Methodology & Assumptions

Failure Mode Definition

  • Top event: SAFE_OUT_PIN = HIGH when platform is NOT mechanically stable
  • Safe state: LOW (de-energized, fail-safe)
  • All paths to false-HIGH enumerated deductively (top-down FTA)
  • All failure modes classified Severity I (Catastrophic)

Root Cause Categories

  • Software logic bug — incorrect gate/threshold conditions
  • RAM / bit-flip corruption — DTCM SEU or latch-up
  • Missing / bypassed gate — threshold not integrity-protected
  • Hardware fault — GPIO, IMU silicon, SPI bus

Key Architectural Finding

  • AND gate (sw_ok × hw_gate) eliminates independent paths
  • hw_gate runs on ASM330LHB Cortex-M0+ (independent silicon)
  • FSM_STABLE_NEED=3 provides 600 ms temporal hysteresis
  • Dominant risk is common-mode (correlated) sensor failure

MIL-STD-882E Standard

  • Table III Risk Assessment Matrix applied
  • Probability: D (Remote) = 10⁻⁶ ≤ P < 10⁻³/hr
  • Probability: E (Improbable) = 10⁻⁸ ≤ P < 10⁻⁶/hr
  • Probability: F (Eliminated) = P ≈ 0 by design
  • PFH estimates assume IEC 61508 SIL 3 operating environment

Recommended Corrective Actions

  • ✓ SPI config-register CRC-8 shadow implemented (CTRL1_XL / CTRL2_G / CTRL3_C / CTRL4_C, 200 ms, poly=0x07)
  • Add SPI sensor data frame CRC — burst reads (12 B at 833 Hz) still unchecked; dominant path gap remains
  • Extend CRC-16 coverage to include spread_max_deg + anchor_max_deg
  • Consider EMI shielding / power supply decoupling (common-mode)
  • Add power-on self-test (POST) for SPI bus integrity

Scope Limitations

  • False-LOW (missed stable event) not analyzed here
  • Quantitative PFH estimates are engineering approximations
  • External EMI / ESD quantification requires site survey
  • Mechanical output stage not in scope
